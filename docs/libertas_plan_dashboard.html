LIBERTAS MIMARI TASARIM GÖRSELİ (RESERVED)

1. Özellik Dağılımı (Feature Breakdown)

F1: Perception Engine

Algı Motoru. Ajanın anlık fiziksel ve içsel durumunu sorgulanabilir bir veri yapısına dönüştürür.

  • Modus.Mind.Perception
  • Modus.World.Spatial
Efor: 3 Gün

F2: Social Graph Query

Sosyal Çizge Sorgusu. ETS tabanlı sosyal ağın LLM bağlamına enjekte edilebilir hale gelmesi.

  • Modus.Social.Insight
  • İlişki raporlama API'si
Efor: 2 Gün

F3: Intent Parser

Niyet Ayrıştırıcı. Kullanıcı mesajlarını ham metinden yapılandırılmış komutlara dönüştürür.

  • Modus.Protocol.Interpreter
  • Chat / Query / Command tespiti
Efor: 4 Gün

F4: Dynamic Context Injector

Dinamik Bağlam Enjektörü. LLM System Prompt'unun statik halden dinamik hale geçişi.

  • Modus.Mind.ContextBuilder
  • Prompt hidrasyonu
Efor: 3 Gün

F5: Feedback Loop

Geri Bildirim Döngüsü. Ajanın eylemlerinin simülasyona yansıması ve sonucun sohbete dönmesi.

  • Modus.Protocol.Bridge
  • Action Dispatcher
Efor: 4 Gün

Hafta 1: Algı ve Anlamlandırma

  • Pzt-Salı: F1 (Perception Engine). Modus.World entegrasyonu.
  • Çarşamba: F2 (Social Graph Query). ETS tablo dönüşümleri.
  • Perş-Cuma: F4 (Dynamic Context Injector). Antigravity ağ geçidi prompt güncellemesi.

Hafta 2: Protokol ve Köprü

  • Pzt-Salı: F3 (Intent Parser). LLM zinciri kurulumu.
  • Çarş-Perş: F5 (Feedback Loop). Simülasyon komut iletimi.
  • Cuma: Entegrasyon testleri, POP 10 yük testi ve Demo.

3. Teknik Mimari (v0.5.0 Flow)

Risk Yönetimi

  • Gecikme (Latency): Çift LLM çağrısı (Intent + Gen) yavaşlatabilir.
    Önlem: Intent için GPT-4o-mini/Flash kullanımı.
  • Halüsinasyon: Ajanın yapmadığı eylemi söylemesi.
    Önlem: Grounding mekanizması ile sıkı bağlama.
  • Token Maliyeti: Context şişmesi.
    Önlem: En yakın 3 obje filtresi.

Başarı Kriterleri

  • Konumsal Farkındalık: "Neredesin?" sorusuna gerçek koordinat (120,50) ve çevre verisi ile yanıt.
  • Sosyal Hafıza: "Ahmet'i tanıyor musun?" -> ETS grafiğinden "Yakın Arkadaş" yanıtı.
  • Eylem-Söz Uyumu: "Dur ve dinlen" komutu -> Simülasyonda Idle state + Enerji artışı.
